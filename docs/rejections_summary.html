Guideline 2.1(b) — Performance
App Completeness (StoreKit Payment Gateway)
Rejection Resolved
Rejection Description
The app rewarded premium features immediately upon tapping subscription packages during testing, without triggering the actual App Store / StoreKit payment flow.
Fix Details
Removed the sandbox dry-run bypass inside the handlePurchase method. If loading available packages fails, the app now properly triggers a standard "Store Connection Error" popup instead of unlocking pro features as a fallback.
App Store Review Note (Copy to paste)
Resolved Guideline 2.1(b): We removed the internal developer sandbox bypass in handlePurchase. The app now strictly interfaces with Apple's StoreKit payment dialog sheet when tapping on our annual or monthly plans. If connection is lost or unavailable, the app correctly throws a Connection Error popup.
Guideline 2.5.4 — Performance
Software Requirements (Background Bluetooth central)
Rejection Resolved
Rejection Description
The app declared support for background Bluetooth (bluetooth-central background mode) in the plist, but no background BLE functionality was located or justified.
Fix Details
Completely removed the bluetooth-central background key from the array of background modes in app.json, restricting background execution exclusively to GPS tracking (location).
App Store Review Note (Copy to paste)
Resolved Guideline 2.5.4: We have completely removed "bluetooth-central" from the UIBackgroundModes key inside the app bundle's Info.plist. Background capabilities are now restricted strictly to "location" for the sole purpose of tracking outdoor exercise routes and workout distance.
Guideline 3.1.2(c) — Business
Payments - Subscriptions (Permanent EULA & Privacy Links)
Rejection Resolved
Rejection Description
The submission did not include EULA and Privacy Policy information within the app. Placing links exclusively on the purchase paywall is insufficient, as subscribers cannot access them after upgrading.
Fix Details
Added a permanent, clean **⚖️ Legal** section card directly inside the **Settings screen** layout. This includes functional links to:
  • Privacy Policy (hosted on GitHub Pages)
  • Apple Standard Terms of Use (EULA)
App Store Review Note (Copy to paste)
Resolved Guideline 3.1.2(c): We have added a permanent Legal category card to our main Settings tab. Users can now access both our Privacy Policy link (https://senthilmkm.github.io/lap-counter/privacy.html) and Apple's Standard EULA terms (https://www.apple.com/legal/internet-services/itunes/dev/stdeula/) at any time, even after completing a premium purchase.
Guideline 4 — Design
User Interface Layout (iPad Air Modal Cut-off)
Rejection Resolved
Rejection Description
The app's paywall overlay modal UI was stretched horizontally and cut off vertically when viewed on an iPad Air 11-inch (M3), making it impossible to read all content or tap the bottom buttons.
Fix Details
Implemented the following layout enhancements:
  • Wrapped the paywall card and workout history summary card in `` containers to support full vertical scrolling on shorter screen viewports.
  • Configured responsive styling that automatically detects iPad/tablet devices: centers the modal card vertically/horizontally and clamps the maximum width to `500pt` instead of stretching to full width.
  • Clamped onboarding step wizard containers to a maximum width of `480pt` on tablet layouts.
App Store Review Note (Copy to paste)
Resolved Guideline 4: We optimized our app's modal layouts for tablets. The paywall card and workout details container now detect iPad devices dynamically to align as a centered card (with a max width of 500pt) instead of stretching horizontally. Additionally, we wrapped the cards inside standard ScrollViews to support scrolling and prevent layout clipping on shorter heights or landscape modes.
